3 Vai su ZipLabs e clicca il bookmarklet

Vai sulla dashboard ZipLabs:
https://ziplabs.teslamotors.com/superset/dashboard/17481/

Fai Force Refresh sulla chart "Tutti gli ordini con telaio in Italia", poi clicca il bookmarklet nei preferiti.

Il bookmarklet legge i dati dal Redux store di Superset (tutte le righe, non solo quelle visibili) e li invia automaticamente alla tab Fleet Viewer via postMessage. Se il Redux store non è disponibile, fa scraping dalla tabella DOM.
Fallback: Se la Fleet Viewer non è aperta in un'altra tab, il bookmarklet scarica un file CSV che puoi caricare manualmente.

3 Clicca il bookmarklet su DRO

Con la pagina DRO aperta e i risultati visibili, clicca il bookmarklet. Estrae: ReferenceNumber, IsEnterpriseOrder, IsScheduled, FinalPaymentStatus, IsContainmenthold, LastKnownVehicleLocation.

Il bookmarklet cerca dati nel DOM e nei framework React/Redux di DRO. I dati vengono inviati alla Fleet Viewer come File 2 (Enterprise) per il merge automatico.
? Come funziona il bridge

I bookmarklet usano window.postMessage() per inviare dati JSON direttamente alla tab della Fleet Viewer, senza bisogno di salvare/caricare file.

  • Il bookmarklet trova la tab Fleet Viewer via window.open()
  • Estrae i dati dal Redux store (ZipLabs) o dal DOM (DRO)
  • Invia un messaggio con type TESLA_FLEET_VIEWER_DATA
  • La Fleet Viewer riceve il messaggio e popola automaticamente gli slot
  • Se la Fleet Viewer non è aperta, scarica un CSV come fallback
  • I dati passano per la stessa pipeline di processAndRender/handleFile2Merge
! Workflow consigliato

Apri 3 tab nel browser:

  • Tab 1: Tesla Fleet Viewer (questa pagina o index.html)
  • Tab 2: ZipLabs Dashboard 17481 → clicca bookmarklet ZipLabs
  • Tab 3: DRO Advisor → clicca bookmarklet DRO
  • Torna alla Tab 1: entrambi gli slot saranno compilati automaticamente
  • Clicca "INIZIALIZZARE SISTEMA" per la dashboard completa
